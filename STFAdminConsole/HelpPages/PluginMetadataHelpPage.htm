Plugin Reference Management

Some Virtual Resources within the Scalable Test Framework use a plugin model for extensibility.  This dialog provide the ability to manage data in the STF database so new plugins can be added and made available for use in test scenarios.  Each plugin is a standalone assembly that conforms to a defined interface.  As new plugins are written and integrated into the framework, they are enabled through this dialog.  Below is a description of the data used to manage plugin references.
  • Name - This is the name of the plugin.  Each plugin assembly uses a naming convention of "Plugin.<name>.DLL", and the name entered in this field must correspond to the name used in the assembly itself.  For example, if a new plugin was created and the assembly name was Plugin.WebAutomation.DLL, then "WebAutomation" would be the name entered here.

  • Title - This defines what the user will see in the dropdown menu when using the STF administration console to add a plugin to a test configuration.  It is the title or label value for the plugin.  Typically it will be very short and may be the same as the Name, but with spaces.  For example, "WebAutomation" as a Name may simply be "Web Automation" for the title. 

  • Group - This field is purely for administrative purposes and provides a way to categorize any plugin that is added to the system.  The field provides a dropdown that shows any existing group names, but also allows the user to enter a new group if needed.  It helps provide an aspect of ownership to the plugin which may be used by testers when they are trying to find the right contact if there is an issue or question with a plugin.

  • Resource Type - This defines what type of Virtual Resource this plugin is associated with.  Some plugins are written to specifically work with a particulary type of Virtual Resource, such as an Office Worker or Admin Worker.  Select the resource that this plugin is associated with.  Future releases of STF will allow the same plugin to be associated with multiple Virtual Resources, but for now you can only associate it with one.
